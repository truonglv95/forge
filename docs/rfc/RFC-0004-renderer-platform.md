# RFC-0004: Native macOS Renderer Platform

## 1. Summary
Forge is an AI-first IDE that prioritizes responsiveness, low input latency, and seamless typography (e.g., Vietnamese, emojis) over immediate cross-platform reach. We propose using **Native macOS Cocoa (AppKit) + CoreText + Metal** via Zig's native C/Objective-C integration, rather than a cross-platform engine like GLFW or Mach, for the M4 MVP.

## 2. Motivation
The MVP is explicitly constrained to macOS. The rendering stack must handle:
- Complex font shaping (ligatures, combining marks, Emojis, RTL).
- Minimal idle CPU/Memory usage (RSS).
- Frame synchronization with the display (ProMotion 120Hz).

Using cross-platform frameworks (e.g., GLFW) typically requires bundling heavyweight shaping libraries like HarfBuzz and FreeType to achieve acceptable typography. By using the OS-native CoreText, we eliminate the need for these libraries, reducing binary size and relying on Apple's highly optimized text shaping.

## 3. Spike Results (M0.2)
We created `tools/renderer-spike`, demonstrating:
1. **Windowing:** A standard macOS window created via AppKit/Objective-C without any C++ or heavy wrapper dependencies.
2. **GPU Surface:** An `MTKView` (MetalKit) attached to the window.
3. **Typography:** CoreText (`CFStringCreateWithCString`) successfully parses and shapes UTF-8 text containing Vietnamese and Emojis natively.
4. **Zig Interop:** Zig effortlessly imports `mac_window.h` and calls into the Objective-C runtime via `@cImport`.

## 4. Decision
We will proceed with the **Native Cocoa + Metal + CoreText** stack for M4 (Native IDE editing foundation).

- **UI & Windowing:** Handled via minimal Objective-C wrappers (`mac_window.m`).
- **Rendering:** Metal, using a glyph atlas generated by CoreText.
- **Event Loop:** Driven by `[NSApp run]` but strictly isolated from Forge's headless execution kernel. File I/O and AI logic will run on background threads, ensuring the render thread never blocks.

## 5. Alternatives Considered
- **Web Technologies (Electron/Tauri):** Disqualified by the "Native IDE" North Star. Too high latency and RSS.
- **Mach Engine / SDL3 / GLFW:** Powerful, but requires compiling and bundling HarfBuzz + FreeType for robust text shaping. On macOS MVP, this is redundant given CoreText's existence.

## 6. Next Steps
Move to M4.1. Add `apps/forge-ide` to `build.zig` using this foundation.
